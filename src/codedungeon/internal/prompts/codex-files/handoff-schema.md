# Handoff Schema

Each phase handoff must include:
- `summary`: one concise paragraph.
- `decisions`: concrete decisions made.
- `artifacts`: files, commands, or outputs produced.
- `traps`: risks, edge cases, or follow-up hazards.
- `open_questions`: unresolved questions.
- `next_input`: exact input needed by the next phase.
- `promise`: `PHASE_<ID>_COMPLETE`.
