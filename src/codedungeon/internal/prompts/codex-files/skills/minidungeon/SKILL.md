# minidungeon

Use for small, well-scoped single-repo changes.

Steps:
- Read the current request, local docs, and relevant code.
- Create a short implementation checklist.
- Make scoped changes with focused tests.
- Run relevant verification.
- Summarize changed files, tests, and residual risk.

Keep DB phase state only when the task has already been bootstrapped into a codedungeon run.
